# Handoff Document

## Description

Generate a comprehensive leave/handoff status document covering all in-progress work by initiative, blocked items, upcoming milestones, and who to contact for what. The one doc that's impossible to create quickly when you need it most.

## Trigger

Run when user says:
- `/handoff`
- "I'm going on leave" / "handoff document" / "coverage doc" / "out of office"

## Workflow

### Step 1: Gather Context

Ask:
```
Handoff details:
1. Duration: [dates you'll be away]
2. Cover person: [who's covering, if known]
3. Special items: [anything critical to highlight]
```

### Step 2: Pull All Active Work

**Jira (via Atlassian MCP):**

```
My in-progress: project in (UKCAUD, DIST, UKCAS) AND assignee = currentUser()
                AND status != Done AND status != Closed ORDER BY priority DESC

My watched: project in (UKCAUD, DIST, UKCAS) AND watcher = currentUser()
            AND status != Done AND status != Closed ORDER BY priority DESC
```

**Local workspace:**
- `workspace/coordinator/weekly-plan.md` → current priorities
- `workspace/coordinator/cross-cutting-concerns.md` → active dependencies
- `workspace/initiatives/*/weekly-todos.md` → in-progress items per initiative
- `workspace/initiatives/*/milestones.md` → upcoming milestones

**Upcoming meetings** (via Playwright if available):
- Meetings during absence period
- Flag any that need the cover person to attend

### Step 3: Build Handoff Document

```markdown
# Handoff Document — Liam Bond

**Away:** {start date} → {end date}
**Cover:** {cover person or "TBD"}
**Generated:** {today}

---

## UKCAUD — In Progress

| Ticket | Summary | Status | Priority | Notes |
|--------|---------|--------|----------|-------|
[All my in-progress UKCAUD tickets]

**Sprint status:** {X} to do, {Y} in progress, {Z} done
**Sprint ends:** {date}
**Key risk:** {any at-risk items}

## DIST — Open Escalations

| Ticket | Summary | Status | Age | Action Needed |
|--------|---------|--------|-----|---------------|
[All my open DIST tickets]

## UKCAS — Items Needing Attention

| Ticket | Summary | Priority | Waiting On |
|--------|---------|----------|------------|
[L3 items I'm responsible for]

## Blocked Items

| Ticket | Blocked By | Owner | Action |
|--------|-----------|-------|--------|
[All blocked tickets across projects]

## Upcoming Milestones

| Date | Milestone | Initiative | Status |
|------|-----------|-----------|--------|
[Milestones within 4 weeks]

## Meetings During Absence

| Date | Meeting | Action Needed |
|------|---------|---------------|
[Meetings that may need coverage]

## Key Contacts

| Area | Contact | Context |
|------|---------|---------|
| UKCAUD sprint | [Scrum master/lead] | Sprint ceremonies |
| DIST escalations | [Platform contact] | Platform issues |
| UKCAS triage | [Support lead] | Customer bugs |
| Goals/Strategy | [Manager] | Strategic decisions |

## Decision Authority

While I'm away, {cover person} can:
- [x] Triage UKCAS L3 tickets
- [x] Move tickets between sprint statuses
- [ ] Create new initiatives (check with me first)
- [ ] Approve scope changes (check with me first)

---

*Generated by /handoff — review and share with {cover person}*
```

### Step 4: Output

Save to: `workspace/coordinator/notes/{today}-handoff.md`

```
Handoff document generated:
- {N} in-progress tickets across {N} projects
- {N} blocked items
- {N} upcoming milestones
- {N} meetings during absence

Saved: workspace/coordinator/notes/{today}-handoff.md

Convert to PDF? (/pdf workspace/coordinator/notes/{today}-handoff.md)
```

# Workspace Search

## Description

Natural language search across the entire workspace — daily logs, decision log, strategy docs, initiative context, archived files, and notes. Answers "when did we decide X?" without manually grep-ing through files.

## Trigger

Run when user says:
- `/search "query"`
- "when did we decide" / "find where we discussed" / "search for"

## Arguments

- `query` (required): natural language search query

## Workflow

### Step 1: Determine Search Scope

Based on the query, determine which files to search:

| Query Type | Files to Search |
|-----------|-----------------|
| Decision/choice | `decision-log.md`, `*/context.md`, daily log |
| Date/timeline | Daily log, milestones, archived weekly plans |
| Ticket/issue | All files (grep for ticket key pattern) |
| Person/stakeholder | All files |
| Strategy/direction | Strategy files, decision log, context files |
| Default | All workspace files |

### Step 2: Search

Read and search across matched files. Use semantic matching against the query — not just keyword grep.

Collect:
- File path
- Matching section/entry
- Date (if from dated entry)
- Context (surrounding text)

### Step 3: Output

```
Found {N} matches for "{query}":

1. [{date}] {file}: {matching excerpt}
2. [{date}] {file}: {matching excerpt}
3. [{date}] {file}: {matching excerpt}
```

If no matches:
```
No matches found for "{query}" in workspace.
Try: broader terms, checking archived files, or searching Jira/Confluence directly.
```
