Overview.

Description.

This project was created to facilitate the efficient updating CaseWare UK's Desktop files along with producing PDF output for comparisons and testing purposes. 
Scope. 

The scope of this project is to update a given CaseWare UK Accounts / Corp Tax engagement while providing options to print the main engagements documents both before and after an update. 
Implementation.

The application is implemented as follows:
The application is launched by the user.

The user makes their selections in the dialog and starts the update process.

The application then creates a list of files that need to be updated. 

Next all of the files to be updated are decompressed. 

Each file is then updated in the manner specified by the selected options.

All files are then re-compressed. 

Internal stakeholders.

Amy Banes

Cheryl Lushey

Phillip Reeves

Adam Martin

Daniela Ana-Luca

Formal change request procedure.

No procedure necessary, feedback and improvements made ad hoc.
Development.

Tools and Environment.

There are a few tools you will need to be familiar with and have installed to get started working on this project, they are as follows: 
Visual Studio 2019 or greater, with packages for .NET cross platform development. 

Git version control software. 

Write Access to the application repositories on Bitbucket. 

A local installation of CaseWare Working Papers, the Templates and engagement files. 

General notes and guidance.

Known Issues.

If CaseWare has been installed but never opened before the updater is run this can cause problems, ensure that a file has been opened in CaseWare at least once before using the app. 

Application deployment, folder structure and dependencies.

Folder Structure.

The application is built as a portable single file .exe with no dependencies, therefore no folder structure, nor any specific installation directory is required to run the app. 
Deployment.

The application should be build following the steps in the repositories readme file. The application should then be digitally signed, a copy then uploaded to SVN in at the location 'CaseWare\_SVN - NOT for Epack Automation\CV Updater Tool' and a copy added to '\\cw-data-1\Development\CaseWare Latest\Internal Applications\CVUpdater'.
Further information.

Please read the http://README.md  files within the aforementioned repository or contact the Author.
Author.

Joe Willson.
